# CRC-100K 数据集上的 UNI 模型评估 - 组会介绍

## 1. 任务目标与技术路线

### 1.1 任务目标
该notebook实现了在CRC-100K数据集上对UNI预训练模型的全面评估，具体目标包括：
- 验证UNI模型在结直肠组织分类任务上的特征提取能力
- 通过多种评估方法（Linear Probe、KNN、ProtoNet、Few-Shot Learning）全面评估模型性能
- 分析不同评估方法和参数设置对结果的影响

### 1.2 数据集
使用CRC-100K ROI数据集，这是一个9类结直肠组织分类任务：
- **训练集**：100,000张图像（11.7 GB）
- **测试集**：7,180张图像（800.3 MB）
- 数据集链接：https://zenodo.org/records/1214456

### 1.3 技术路线
```
加载预训练UNI模型 → 提取图像特征 → 多种评估方法评估特征质量
```

#### 1.3.1 模型配置
UNI模型是一个高性能的Vision Transformer模型，具体配置如下：
- 模型类型：Vision Transformer
- 特征维度：1024
- Patch size：(16, 16)
- 图像尺寸：(224, 224)
- Transformer层数：24
- 注意力头数：16
- 总参数量：303,350,784

#### 1.3.2 功能模块
1. **模型加载与配置**：通过`get_encoder`函数加载预训练UNI模型和对应的图像变换
2. **数据加载**：使用`torchvision.datasets.ImageFolder`加载CRC-100K数据集
3. **特征提取**：通过`extract_patch_features_from_dataloader`函数提取图像特征
4. **线性探针评估**：使用`eval_linear_probe`评估线性分类器性能
5. **KNN评估**：使用`eval_knn`评估K最近邻分类器性能
6. **ProtoNet评估**：基于原型网络进行分类评估
7. **少样本学习评估**：使用`eval_fewshot`评估少样本学习场景下的性能

## 3. 实验过程中遇到的问题与解决方案

### 3.1 特征提取效率问题
- **问题**：100K训练图像的特征提取耗时较长，且可能导致内存溢出
- **解决方案**：
  - 使用批量处理（batch_size=256）
  - 利用GPU加速计算
  - 采用多线程数据加载（num_workers=16）

### 3.2 线性探针训练优化
- **问题**：大特征维度下的线性分类器训练效率低
- **解决方案**：
  - 使用L-BFGS优化算法
  - 设置合适的正则化强度（C=特征维度×类别数/100）
  - 限制最大迭代次数（max_iter=1000）

### 3.3 少样本学习中的样本选择偏差
- **问题**：随机选择支持样本可能导致类别不平衡
- **解决方案**：
  - 使用`FewShotEpisodeSampler`确保每个类别选择相同数量的样本
  - 多次采样取平均值减少随机性影响

### 3.4 KNN计算效率问题
- **问题**：大规模特征空间下的KNN计算缓慢
- **解决方案**：
  - 使用scikit-learn的KNeighborsClassifier
  - 启用多线程计算（n_jobs=8）

## 4. 不同参数设置对实验结果的影响

### 4.1 特征处理策略的影响
| 策略 | 对性能的影响 |
|------|--------------|
| 特征中心化 | 显著提高分类性能，减少数据分布偏差 |
| 特征归一化 | 提高KNN和ProtoNet的性能，使特征具有可比性 |
| 特征平均 | 对ProtoNet和少样本学习至关重要，减少噪声影响 |

### 4.2 KNN中K值的选择
- **较小K值**：对噪声敏感，但能捕捉局部模式
- **较大K值**：更稳定，但可能忽略局部模式
- **实验选择**：K=20，在性能和稳定性之间取得平衡

### 4.3 少样本学习中Shot数量的影响
- **较少Shot**：性能下降明显，模型难以学习类别原型
- **较多Shot**：性能提升，但接近全样本学习
- **实验选择**：16-shot，模拟实际少样本场景

### 4.4 线性探针中的正则化强度
- **较强正则化**：防止过拟合，但可能欠拟合
- **较弱正则化**：更好拟合训练数据，但可能过拟合
- **实验选择**：C=特征维度×类别数/100，自动适应不同数据集

## 5. 局限性及未来可改进方向

### 5.1 局限性
1. **单一数据集评估**：仅在CRC-100K数据集上进行了评估，缺乏跨数据集的泛化性验证
2. **固定评估协议**：使用了标准的评估协议，未探索更多的特征融合和增强方法
3. **计算资源需求高**：处理100K图像需要大量计算资源和时间

### 5.2 未来改进方向
1. **跨数据集验证**：在更多的组织病理学数据集上评估模型性能，如Camelyon16、PAIP等
2. **特征工程与增强**：
   - 尝试特征融合策略（如多尺度特征融合）
   - 探索数据增强方法提高模型鲁棒性
3. **模型轻量化**：
   - 探索知识蒸馏技术压缩模型
   - 设计更高效的轻量级架构
4. **领域适应**：研究如何将UNI模型适配到更多特定的病理学任务中
5. **自监督学习改进**：
   - 探索更适合病理学图像的自监督学习目标
   - 研究更长时间的预训练对性能的影响

## 6. 总结

本notebook在CRC-100K数据集上全面评估了UNI预训练模型的性能，通过多种评估方法验证了其在结直肠组织分类任务上的有效性。实验结果表明：

1. UNI模型提取的特征具有很强的判别能力，在各种评估方法下都取得了优异的性能
